Examine the cake

In this tutorial, we will be using JSON properties, destructuring, nested destructuring, arrow functions, objects, and aliases to take a look at our sweet, sweet cake. The advanced section expect you to have finished the basic section and its challenges. Concepts described there won't be revised here.

Concepts Used

Destructuring

As covered in the basic section with arrays, we can use destructuring to easily extract data from an array/piece of JSON data, but the syntax for JSON destructuring is different.

const describeSweetnessOfACake = ({sweetness:s}) => { console.log(`sweetness of a cake: ${s}`) }

The curly brackets tell us we are accessing a JSON object from a variable given by the user. The colon used is for an alias, so we can refer to the JSON object sweetness as s in our code to save time.

Nested destructuring

Nested destructuring is similar to regular destructuring except we extract data from a sub-section in a JSON object (for example, the object address inside the object factory requires nested destructuring to be accessed).

const printAddress = ({factory:{address :{street:s,zip:z}}}) => { console.log(s, z) }

After the colons we specify, a curly bracket specifies we are destructuring another JSON object. Once you have started nesting, it isn't recommended to use aliases unless you're sure you're not going to nest more (for example, for the JSON object address here) because then we cannot continue our nesting process, forcing us to start nesting later in the program. At the end nested object we specify, we can then add our aliases and use them in our code.

JSON Object Access Types

You can access JSON objects in many different ways.

// Method 1 (won't work with underscores, special characters, or variables containing a property value):

console.log(cake.decorations)

// Method 2 (can be used with text-based and integer-based access, the latter not recommended, and works with underscores and some special characters):

console.log(cake["decorations_test"])

// Method 1 - Nested Object Access:

console.log(cake.factory.address.zip)

// Method 2 - Nested Object Access:

console.log(cake["factory"]["address"]["zip"])

Unlike functions, we must specify the parent of the JSON data we will be accessing (in this case, the cake constant. There are 2 well-known ways as shown above to access JSON objects. We will be using both here.

Examine the cake

Now to mesh these concepts together.

First, we define a constant containing our JSON data. It has various properties we will be accessing via destructuring and regular object access. We then access a few JSON properties and print them to the console just for learning purposes. Then comes our first arrow function called describeCake, which we will use to output the entire variable cake as JSON data in the console with destructuring. Then we create a function (also an arrow one) to describe the sweetness of our delicious cake. We then take the sweetness property (curly brackets specify we are accessing JSON data and since we don't have any other JSON variables it defaults to cake) and create an alias for it using a colon. This alias is described as s. Our next function is the first to demonstrate nested destructuring, as we access the street and zip objects from the address object inside the factory object. We then create aliases (s and z respectively) for the street and zip objects and print them in the console. At the end, we execute all three arrow functions we created and give the variable cake as our variable containing JSON. That's it!

const cake = { decorations_test: ["cherry", "name"], sweetness: 10, rating: 10, factory : { address : { street: "123 Internet Boulevard", zip: "12345" } } } console.log(cake.decorations_test)

// Remember, underscores don't work with method 1, use method 2 instead!

console.log(cake["decorations_test"]) console.log(cake.factory.address.zip) const describeCake = (cake) => { console.log(cake) } const describeSweetnessOfACake = ({sweetness:s}) => { console.log(`sweetness of a cake: ${s}`) } const printAddress = ({factory:{address :{street:s,zip:z}}}) => { console.log(s, z) } describeCake(cake) describeSweetnessOfACake(cake) printAddress(cake)
